# 智能家居多智能体控制 · 路演视频脚本

- **总时长**：约 120 秒（2 分钟）
- **结构**：实景开场（15s）+ 三个交互用例（90s）+ 花絮（15s）
- **核心展示对象**：阳光房实景 + 大屏数字孪生 + 小机器人 + 多智能体讨论文本
- **设备/Agent 映射**：
  - `agent 1` → 空调 `ac.main`
  - `agent 2` → 灯光 `light.perimeter` / `light.entry`
  - `agent 3` → 电脑屏幕 `screen.main`

---

## Part 1 · 实景开场（0:00 – 0:15，约 15s）

### 0:00 – 0:06 ｜ 阳光房外景推进
- **画面**：从阳光房外侧低角度推镜，玻璃顶反光、绿植入画，最后定格门口/小机器人在门厅待命的全景。
- **字幕**（左下角小字）：`AttraX · Sunroom Living Lab`
- **旁白**（女声/男声任一，平稳）：
  > "这是我们的阳光房——一间会自己思考的房子。"

### 0:06 – 0:15 ｜ 进入室内 + 设备点出
- **画面**：从门口缓推到桌前，依次扫过空调、灯、屏幕、小机器人、桌面。每扫到一个设备，画面右上角弹出小标签 `空调 / 灯 / 屏幕 / 小管家`，叠加 0.3s 高光描边。
- **大屏内容**：数字孪生 3D 模型同步亮起，3 个 agent 卡片（A1 空调 / A2 灯 / A3 屏幕）默认 `rest` 灰色。
- **旁白**：
  > "三类设备、三个智能体，背后是一个会自己讨论的多智能体系统。"

## Part 2 · 三个交互用例（0:15 – 1:45，约 90s · 每段 30s）

> 每个用例的统一节奏：
> 1. **0–5s**：用户对小机器人说一句自然语言（不带具体设备指令）
> 2. **5–18s**：大屏左侧浮出"多智能体讨论"对话框，3 个 agent 各发一句话，A1/A2/A3 头像点亮变 `work`
> 3. **18–25s**：3D 模型 + 实景设备同步变化，受影响设备出现脉冲高光环 + 浮动 badge（如 `→ 24°C`、`+30%`）
> 4. **25–30s**：机器人复述决议一句

### 用例一 ｜ 客户来访 · 软约束推理（0:15 – 0:45）

- **场景设定**：人物坐在桌前，看了眼时间，对小机器人讲话。
- **0:15 – 0:20** 用户对白：
  > "再过十分钟客户要来看 demo，帮我准备一下。"
- **0:20 – 0:33** 大屏讨论文本（逐句出现）：
  - `[A1 空调]`：现在 28°C 偏闷，客户来访建议提前制冷到 24°C，避免会议时启停噪音。
  - `[A2 灯光]`：环境光偏暗，开周圈灯到 65%、入口灯打开，让客人一进门就被引导到桌前。
  - `[A3 屏幕]`：切到汇报模式，标题"演示已就绪"，关闭通知。
- **0:33 – 0:42** 实景 + 数字孪生联动：
  - 空调风口 LED 蓝色亮起，风扇转动（或音效辅助）
  - 灯一组先亮、入口灯延迟 0.5s 亮（错位感更"有人住"）
  - 屏幕切到深蓝汇报底色
  - 大屏 3D 模型上对应设备出现蓝/黄/青脉冲环
- **0:42 – 0:45** 机器人语音：
  > "好啦，10 分钟内调到汇报模式。"
- **关键看点**：用户**没说**任何具体设备，是 agents 自己拆解出"客户/演示/十分钟"这些软约束并各自决策。

### 用例二 ｜ 模糊体感指令 · 多 agent 协商（0:45 – 1:15）

- **场景设定**：人物揉了揉眼睛，靠在椅背上。
- **0:45 – 0:50** 用户对白：
  > "感觉有点闷，但又怕开空调直吹。"
- **0:50 – 1:03** 大屏讨论文本（这次显式带"反驳/协商"，体现讨论价值）：
  - `[A1 空调]`：温度 27.5°C 接近闷热阈值，提议制冷 25°C，**风速调到最低、风向上吹**。
  - `[A2 灯光]`：可以同步把灯光色温从 4200K 拉到 3500K，降低视觉热感，顺手帮 A1 一把。
  - `[A1 空调]` （第二轮）：同意，那我先 25°C 试 5 分钟，不行再降 1°C。
- **1:03 – 1:12** 实景 + 数字孪生联动：
  - 空调启动，风口角度向上偏（如有舵机）/或屏显图标
  - 灯光颜色肉眼可见地"暖"了
  - 大屏 3D 模型空调外壳叠加蓝色冷感 tint，灯光颜色随 cct 实时偏移
- **1:12 – 1:15** 机器人语音：
  > "先这样试试，5 分钟后我再问你。"
- **关键看点**：**两个 agent 跨设备协商**——这正是多智能体相对单 LLM 的差异点。

### 用例三 ｜ 离家延伸 · 跨域协同（1:15 – 1:45）

- **场景设定**：人物拿起包准备出门。
- **1:15 – 1:20** 用户对白：
  > "我出去接个人，半小时就回。"
- **1:20 – 1:33** 大屏讨论文本：
  - `[A2 灯光]`：周圈灯调到 8% 留个引导亮度，入口灯熄灭。
  - `[A1 空调]`：保持 25°C 但切到节能模式，半小时回来不重启更舒服。
  - `[A3 屏幕]`：屏幕进入待机黑屏，保留汇报内容草稿。
  - `[+ 扩展卡片]` 一个浅灰卡 `[车辆充电桩 (扩展)]`：检测到您出行 30 分钟，已为您预热车辆并显示充电进度 78%。
- **1:33 – 1:42** 实景 + 数字孪生联动：
  - 灯渐暗（不要瞬切，要 1.5s 缓动，更"住宅感"）
  - 屏幕黑屏
  - 大屏 3D 模型最下方滑入一个**汽车充电状态浮卡**，作为非家居扩展展示点
- **1:42 – 1:45** 机器人语音：
  > "路上慢点，回来给你恢复。"
- **关键看点**：**节能/安全的硬约束 + 跨域设备扩展性**——回应路演时关于"是否能接非家居设备"的问题。

## Part 3 · 花絮（1:45 – 2:00，约 15s）

- **节奏**：5 个 1.5–2.5s 的短切镜头，配轻快节拍音乐，画面带轻微抖动/手持感。
- **镜头列表**：
  1. **1:45 – 1:48** 团队凌晨调代码，屏幕反光在脸上
  2. **1:48 – 1:50** 小机器人第一次"翻车"原地转圈，团队大笑
  3. **1:50 – 1:53** 第一次成功联动那一刻三人同时看大屏
  4. **1:53 – 1:55** 阳光房白天空镜（茶杯、笔记本、阳光投影）
  5. **1:55 – 2:00** 全员合影 + 字幕

- **结尾字幕**（黑底白字 1.5s）：
  ```
  AttraX × HomeLLM
  让房子先开始思考
  成员姓名
  ```
- **片尾旁白**（可选，1 句）：
  > "下一步——让每个空间都长出自己的智能体。"

## 拍摄与制作清单

### 现场需要准备的硬件
- [ ] 阳光房现场打扫 + 桌面布景（笔记本、咖啡杯、绿植）
- [ ] 小机器人充电完成、SSID 已连阳光房网络
- [ ] 大屏 1080p 接出，运行 `digital_twin frontend` + 后端 mock_backend
- [ ] 灯（perimeter + entry 两组）能受 API 控制，**或现场用绿米/可控插座代替**
- [ ] 空调若不可控，准备**风扇 + 提示音效**冒充启动反馈

### 软件状态确认
- [ ] `backend/.env` 已切到 `deepseek-v4-pro`（OPENAI_MAX_TOKENS=4096）
- [ ] `mock_backend` 在 `:8787`，前端 SSE 连通
- [ ] **设备高亮脉冲环**已实装（lastChangedAt → 脉冲 emissive 环 + 浮动 badge）
- [ ] 多智能体讨论文本面板在前端讨论区可见

### 拍摄建议
- **镜头**：手持 + 一只滑轨，至少 1 个广角覆盖全屋、1 个特写覆盖设备/屏幕
- **录音**：用户对话 + 机器人语音必须**同期声**清晰，环境噪音降到最低
- **大屏拍摄**：相机快门同步到 50/60Hz 避免摩尔纹；屏幕亮度调到 70% 防过曝
- **后期**：每个用例的"3 个 agent 发言"做**逐句字幕烧录**，避免观众漏看

### 三个用例 prompt（直接喂给 deepseek-v4-pro 即可复现）
1. `等下客户要来看 demo，帮我准备一下`
2. `感觉有点闷，但又怕开空调直吹`
3. `我出去接个人，半小时就回`

## 兜底方案（如机器人/硬件现场失败）

| 失败项 | 兜底 |
|--------|------|
| 小机器人不响应 | 改用桌面 chatbot UI 输入文本，旁白说"机器人是我们的语音入口" |
| 空调/灯无法实控 | 全部走数字孪生展示，旁白措辞改为"在数字孪生上同步执行" |
| 大屏掉线 | 改放 30s 提前录的纯软件 demo 视频，仍保留 3 段用例结构 |
| 多 agent 讨论延迟 > 15s | 提前把 3 个用例的 LLM 输出**预录成视频片段**插入剪辑，现场复述即可 |
