# Dataset Eval Slide Notes

## 页标题
当前数据集验证结果：系统已跑通，但与标签动作尚未对齐

## 页面要点

### 一句话结论
当前系统已经具备稳定执行和较低时延，但在离线数据集评测中，最终动作与标准标签的语义对齐仍然较弱。主要问题不是“系统跑不起来”，而是“系统会做过多决策，而且这些决策没有准确落在数据集定义的设备域和动作空间上”。

### 讲稿
这一页展示的是我们把当前多 agent 中枢节点，直接放到 episode 级数据集上做离线评测后的结果。

首先，从工程角度看，当前系统已经可以稳定跑通。`execution_success_rate` 达到 1.0，平均延迟大约 65.8 毫秒。这说明我们的中枢框架在执行链路和实时性上已经具备可用性，后续实验的瓶颈不再是系统是否能运行，而是系统是否能做出正确决策。

第二，从行为层面看，系统当前存在明显的过度唤醒和过度生成。平均每个任务会唤醒 2.98 个 agent，但真实标签平均只需要 1.84 个动作；同时系统平均会输出 4.64 个动作，显著高于真实动作数。这使得动作数量误差达到 3.04，说明系统更像是在“扩展场景”，而不是“输出最小必要动作集合”。

第三，从正确性角度看，目前最大的短板是语义对齐。`wakeup_agent_f1` 只有 0.263，说明一开始就没有稳定选对该参与讨论的 agent。更关键的是，`proposal_domain_f1`、`final_domain_f1` 和 `final_action_f1` 都为 0，说明 agent 提案以及最终裁决，在设备域和完整动作层面都还没有和数据集标签形成一致映射。不过，`final_service_f1` 仍然有 0.373，这说明系统并不是完全不会决策，而是在动作意图层面已经有一定信号，只是还没有正确映射到目标设备和标准动作空间。

所以，这一页的核心结论是：当前系统已经完成了“工程可运行”的阶段，下一阶段的重点应该转向三件事，分别是 agent 唤醒策略优化、动作裁剪机制优化，以及本地设备语义与数据集 canonical action space 的统一映射。
